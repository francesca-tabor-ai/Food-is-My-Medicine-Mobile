## 1. Introduction
The Account System is the foundational layer of the "Food is My Medicine" app. It handles user authentication and collects essential demographic data required for accurate biomarker interpretation.

## 2. Goals
- Provide a frictionless sign-up and login experience.
- Securely store user profile data.
- Collect critical biological data (age, sex, height, weight) for personalized medical logic.

## 3. Functional Requirements
### 3.1 Authentication
- **Social Sign-up/Login:** Support for Apple and Google OAuth.
- **Email Sign-up/Login:** Standard email and password authentication.
- **Session Management:** Maintain secure user sessions.

### 3.2 User Profile
- **Mandatory Fields:**
    - Name
    - Age (integer)
    - Sex (Male/Female/Other - critical for biomarker reference ranges)
    - Country (to determine local units and food availability)
    - Height (cm or ft/in)
    - Weight (kg or lbs)

## 4. Technical Constraints
- Built using **Supabase Auth** for authentication.
- Profile data stored in a Supabase `profiles` table.

## 5. Success Metrics
- Conversion rate from app install to completed profile.
- Percentage of users using social vs. email login.
